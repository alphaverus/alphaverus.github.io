Abstract

Automated code generation with large language models (LLMs) has shown promise, but correctness is not guaranteed. Formal verification can provide mathematical guarantees of correctness, yet verified code generation is hindered by data scarcity and the complexity of proofs. AlphaVerus is a self-improving framework that bootstraps formally verified code generation by iteratively translating verified programs from a resource-rich language (Dafny) into a target language (Verus, a Rust-based verifier). AlphaVerus employs a three-phase process: exploration of candidate translations, Treefinement‚Äîa novel tree search for iterative refinement guided by verifier feedback, and a critique step that filters out trivial or misaligned specifications. Through this iterative process, AlphaVerus improves itself without human intervention or model fine-tuning, and enables a LLaMA-70B model to solve verified versions of standard coding benchmarks. Our results open the door to more trustworthy code-generation agents.

Results Summary

AlphaVerus achieves significant accuracy gains on verified code benchmarks compared to baseline LLMs and GPT-4-based methods.

Using AlphaVerus

AlphaVerus provides a few-shot prompting methodology and utilities to incorporate formal verification into your code generation workflows. It does not require model fine-tuning and integrates seamlessly with open-source LLMs.


1. Installation

pip install alphaverus

2. Run AlphaVerus

from alphaverus import AlphaVerus

verifier = AlphaVerus()
verifier.run_translation(source_data)
results = verifier.evaluate(test_specs)
